Protocol Summary

The project will create a quantitative, source-linked map of how research integrity systems are organized, governed, implemented, and made transparent across Europe.

Protocol premise. Research integrity structures can be studied as population-level systems. The project treats countries, institutions, and governance bodies as units of observation, applies explicit eligibility criteria, records system characteristics, and quantifies outcomes such as transparency, independence, procedural detail, and public reporting.

Rationale

European research integrity policy is extensive but fragmented. A source-linked quantitative map can make differences in governance, accountability, prevention, investigation, and transparency visible in a form that supports comparative analysis.

Problem Statement

Research integrity systems are often described through narrative country reports, policy statements, codes of conduct, and institutional webpages. These materials vary in language, legal status, and level of detail.

This project will convert heterogeneous source material into a versioned dataset with operational definitions, extraction rules, adjudication procedures, and confidence ratings.

Clinical Epidemiology Viewpoint

The protocol borrows from systematic review methods: predefined eligibility criteria, calibration exercises, dual extraction for high-risk variables, source hierarchy, reproducible coding, missingness labels, and transparent audit trails.

Instead of patients, exposures, and clinical outcomes, the units are countries, organizations, rules, procedures, and governance outputs.

Objectives

The study will estimate, describe, and compare the observable architecture of research integrity systems across Europe.

Primary Objective

To map the existence and characteristics of national and institutional research integrity structures across European countries using source-linked quantitative indicators.

Scope and Definitions

The project will use explicit inclusion criteria to avoid drifting between research integrity, research ethics, compliance, quality assurance, and general academic governance.

Research integrity structure means the formal and observable system that defines responsible conduct of research, prevents breaches, handles concerns, investigates misconduct, records outcomes, and communicates standards to researchers, institutions, funders, and the public.

The intended final frame is Europe as a broad region. The first analytic release can use a staged frame to protect feasibility: EU member states, EEA countries, Switzerland, and the United Kingdom first; wider Council of Europe countries can be added after piloting and translation capacity are confirmed.

Study Design

A descriptive, cross-sectional mapping study with longitudinal versioning and repeatable source verification.

Define UnitsConfirm country frame, organization classes, and analytic hierarchy.
Find SourcesSearch official, legal, institutional, funder, and published sources using recorded strategies.
Extract DataCode variables with source identifiers, quotes, dates, and uncertainty labels.
AdjudicateResolve disagreements, document decisions, and update the codebook.
AnalyzeProduce comparative tables, completeness maps, typologies, and sensitivity checks.

Target Estimands

The protocol does not estimate causal effects. It estimates descriptive system parameters, such as the proportion of countries with a national integrity body, the proportion with public case reports, the median number of procedural safeguards visible in policy, and the distribution of source confidence across domains.

Repeat Measurement

Each country and organization record should have a version date. When laws, policies, or websites change, the prior value remains in the archive and a new observation is added with an effective date, access date, and change note.

Data Model

The data model separates entities, observations, sources, variables, and extraction decisions so that every reported value can be audited.

Entity Tables

Countries, national bodies, ministries, funders, research-performing organizations, integrity committees, ombuds offices, and policy documents should be stored as separate entities with stable identifiers.

Observation Tables

Each variable value should be stored with entity ID, variable ID, value, unit, source ID, extraction date, extractor ID, confidence rating, and notes. This avoids hiding evidence inside free-text summaries.

The source strategy will prioritize primary, official, current, and archived materials. Secondary literature can inform searches but should not replace direct evidence when coding system variables.

Required Source Fields

Every source record should include source ID, entity ID, title, issuing body, source tier, URL or archival location, publication date when available, access date, language, machine translation status, extractor ID, and a short evidence note.

Missingness Labels

Missing data should not be coded as absence. Suggested labels: not_found, not_applicable, unclear, source_inaccessible, translation_pending, and requires_expert_confirmation.

Quality Assurance

Quality control should target the main risks in this project: language error, obsolete sources, hidden policy pages, ambiguous legal status, and inconsistent judgement between extractors.

Extractor Calibration

Before full extraction, each extractor should independently code the same pilot countries. Agreement should be reviewed by variable, not only overall. Variables with repeated disagreement need clearer definitions or examples in the codebook.

Evidence Confidence

Confidence should combine source tier, source recency, directness of evidence, translation reliability, and consistency across sources. Confidence is attached to each observation, not to a country as a whole.

Analysis Plan

The primary analyses will be descriptive and comparative. Any index scores should remain transparent, decomposable, and source-linked.

Descriptive Analyses

Summarize categorical variables as counts and proportions across countries and organization types. Summarize ordinal scores with medians and ranges. Report missingness by variable, country, language, source tier, and entity type.

Composite Measures

Composite scores can be useful for visualization but should not replace component variables. Candidate indices include transparency, procedural completeness, prevention infrastructure, institutionalization, and public accountability. Each index must have a prespecified scoring rule and sensitivity analysis.
